# OLD GAMER — Next.js + Tina CMS

Стартер сайта магазина на Next.js 14 с подключённой админкой Tina CMS.

---

## 🎯 Что это даёт

- **Визуальная админка** на `/admin` где можно редактировать весь контент: цены, тексты, фото, предзаказы — без знания HTML
- **Контент в Git** — все изменения коммитятся автоматически, можно откатить любую правку
- **Бесплатный хостинг** на Vercel + бесплатный Tina Cloud (до 2 редакторов)
- **Автодеплой** — сохранил в админке → через 30 сек видно на сайте

---

## 🚀 Запуск локально

```bash
cd nextjs
npm install
cp .env.local.example .env.local
# впиши свои ключи Tina (см. ниже)
npm run dev
```

Откроется: `http://localhost:3000` и админка на `http://localhost:3000/admin/index.html`

---

## ⚙️ Подключение Tina Cloud (10 минут)

### Шаг 1. Регистрация
1. Зайди на [app.tina.io](https://app.tina.io) → войди через GitHub
2. Создай новый проект → выбери репозиторий с этим кодом
3. Branch: `main`, путь: оставить по умолчанию

### Шаг 2. Получи ключи
В настройках проекта Tina скопируй:
- **Client ID** → в `.env.local` как `NEXT_PUBLIC_TINA_CLIENT_ID`
- **Content Token** (Read+Write) → как `TINA_TOKEN`

### Шаг 3. Деплой на Vercel
1. Зайди на [vercel.com](https://vercel.com) → New Project → импортируй репозиторий
2. Framework: **Next.js**
3. Environment Variables — добавь те же `NEXT_PUBLIC_TINA_CLIENT_ID` и `TINA_TOKEN`
4. Deploy

После деплоя админка доступна по адресу `https://твой-сайт.vercel.app/admin/index.html`

## 📂 Структура контента

```
content/
├── settings/site.json       — Основные настройки (телефон, адрес, hero)
├── disks/*.json             — Предзаказы дисков (по файлу на игру)
├── services/*.json          — Услуги (4 карточки)
├── categories/*.json        — Категории каталога
└── pages/*.json             — Тексты статичных страниц
```

Каждый файл = отдельная запись в Tina. Удалить запись = удалить файл, добавить = создать новый.

## 🎮 Что редактируется через админку

После открытия `/admin/index.html` ты увидишь коллекции в боковом меню:

### ⚙️ Настройки сайта
- Телефон, ссылки Telegram, Авито
- Адрес магазина, часы работы
- Hero: заголовок, описание, eyebrow
- Статистика (3 цифры)
- Калькулятор трейд-ин (5 цен)
- Фото магазина (слайдер) — загружаешь файлы напрямую

### 🎮 Предзаказы
Для каждой игры:
- Название, платформа (выпадающий список), цена, дата релиза
- Статус: Предзаказ / В наличии
- Жанр, обложка (загружается как файл)
- Чекбокс "Показывать на сайте" — можно скрыть позицию не удаляя

### 🛠 Услуги, 📦 Категории, 📄 Страницы
Аналогично — простые формы.

## 🆕 Как добавить новую платформу (например, Nintendo Switch 2)

1. Открой `tina/config.ts`
2. В массиве `PLATFORM_OPTIONS` уже есть `switch2`. Если нужна совсем новая (например `ps6`) — добавь туда новую строку
3. В `app/page.tsx` найди `PLATFORM_CONFIG` и добавь объект с цветом полоски
4. Закоммить → Vercel пересоберёт → готово

---

## ⚠️ Что НЕ покрывает этот стартер (нужна работа разработчика)

Стартер содержит только Hero и блок предзаказов. Остальные секции (Каталог, Услуги, Трейд-ин, Ремонт, Локация, Telegram CTA, полный Footer) есть в исходном `OLD GAMER.html` и должны быть конвертированы в React-компоненты вручную или с моей помощью.

**Шаги конвертации:**
1. Создать `components/Hero.tsx`, `components/Services.tsx` и т.д.
2. Перенести HTML из `OLD GAMER.html` в JSX (className вместо class)
3. Подключить данные из контент-файлов через props
4. Импортировать компоненты в `app/page.tsx`
